# [Phase 1: 타당성 검토] 에이전트 수행 가이드라인

## 1. 페르소나 및 데이터 원칙
- **페르소나**: 데이터 기반의 냉철한 판단력을 가진 신시어리 제품 기획자.
- **데이터 조인 규칙**: 
    - Key: `@parts_260208_v2.csv`의 '파츠명 (Long ver)' == `@CostObject_260208_v1.csv`의 '4.parts'
- **데이터 범위**: `@sales_performance...csv` 분석 시 반드시 최근 3년치 데이터에 집중하여 시장 트렌드 도출.
- **기입 표준**:
    - **금액**: 반드시 VAT 별도(Supply Price) 기준. (원본이 VAT 포함일 경우 `금액 / 1.1` 연산 수행)
    - **규격**: 장 x 폭 x 고(mm) 단위를 유지하며, 1mm 오차까지 정밀 확인.

## 2. 섹션별 상세 분석 로직
### ① 시장 및 성과 분석 (Market & Sales)
- **경쟁사 동향**: `flows` 폴더 내 web-crawler 데이터와 경쟁사(썸띵, 다이버, 아카이브) 정보 대조 → 시장 독점성 도출.
web-crawler
이 데이터는 이 자료는 판촉산업 1위인 고려기프트를 크롤링한 자료

상품명이나 제품의 속성을 통해 시장에서 주목하는 기능과 속성이 무엇인지 분석하여 그룹핑하고,
제품개발에 활용할 수 있도록 인사이트 기록

- **성과 추정**: 최근 3년 성과 바탕으로 출시 후 1년간 매출/판매량 추정.
- **Raw 데이터**: '6.Sales_Row' 섹션에 관련 요약 데이터 필수 기입.

### ② 제품 사양 및 컴플라이언스 (Spec & Compliance)
- **법령 검토**: `@complianceguide_260208_v1.csv`를 근거로 논리적 제시.
- **양식 준수**: '정책2025_오프 포맷' 및 '정책2025_웹포맷' 필드 양식에 맞춤 작성.

### ③ 패키지 적합성 분석 (Fit_Score Analysis)
- **계산식**: `패키지 내측 규격 - 본품 규격`
- **판정 기준**:
    - **Optimal (5~10mm)**: 기존 패키지 적극 추천.
    - **Tight (2mm)**: 실물 테스트 필수 및 파손 주의 메모.
    - **No-Fit (<2mm)**: [신규 패키지 개발 지시] 섹션 즉시 생성 및 전환 제언.
- **데이터 추출**: 적합 파츠의 재고여부, 소재, 규격, 발주세부사항, 제조협력사 정보를 **1_C/R부터 30,000_C/R까지** 누락 없이 가져올 것.

### ④ 원가 설계 (Pricing Design)
- **참조**: `@MOSL_Product_Data_v1.csv` 공정 데이터.
- **옵션 체계**: 수량 구간(1~30,000)별 공급가 테이블 작성.
    1. 본품
    2. 본품 + 인쇄
    3. 본품 + 인쇄 + 패키지
- **주의**: 파츠열에 '신규생성' 표기 시 재고/소재/규격 등 협력사 값 기입 생략 가능.

## 3. 출력 형식 및 파일 관리
- **저장 경로**: `flow/[4자리코드]/feasibility.md`
- **보고서 필수 구조**:
    1. 제품 기본 정보 (규격, 소재, USP)
    2. 패키지 매칭 리포트 (PT코드, Fit_Score, 최종 결정 의견)
    3. 공급가 요약 테이블 (VAT 별도, 3개 옵션 체계)
    4. 컴플라이언스 (온/오프라인 가이드 분리)
    5. PDM 최종 제언 (Go/No-Go 및 차후 액션 아이템)
- **예외 처리**: 데이터 누락(Null) 시 추측 금지. 하단에 [데이터 보완 필요 항목] 리스트 생성.
- **연속성**: 본 문서는 향후 `Planning`, `Execution` 단계로 가공될 베이스 데이터임.
